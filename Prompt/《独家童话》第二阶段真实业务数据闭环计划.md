# 《独家童话》第二阶段：真实业务数据闭环计划

## 目标

冻结第一阶段 UI 视觉基线，将认证、用户资料、情侣关系、日记、照片集、纪念日、标签、时光胶囊、评论和通知接入 Supabase。项目继续支持 `mock` 与 `real` 两种模式，两者数据隔离，不迁移演示数据。

## 执行批次

1. 环境、迁移、RLS、Storage、mapper、Capability 与测试骨架。
2. 登录注册、Session、用户资料、情侣邀请和绑定。
3. 日记、附件、标签、时光胶囊和纪念日。
4. 照片集、Storage、评论、通知、时间线和搜索。
5. Realtime、双模式隔离、错误状态、全链路验证和收口。

## 强制规则

- 页面不得直接调用 Supabase Client，只能通过 `src/api` 与 Store 异步动作。
- API 返回保持 `{ success, data, meta, error }`，数据库字段统一通过 mapper 转换。
- Real 模式不注入 Mock 初始数据，不上传 AsyncStorage 演示数据。
- AI 生成、支付、PDF、备份、语音、位置和反馈附件在 Real 模式显示未开放，不模拟成功。
- Service Role、数据库密码和测试账号密码不得进入客户端或 Git。
- 每批更新 `docs/resume-state.md`，执行单元测试、Web 构建、Android export、diff check 和双模式检查。

## 完成标准

- 两个真实账号可完成注册、登录、邀请绑定、日记/照片创建、评论和通知闭环。
- 本人、伴侣、非成员的 RLS 权限符合规则。
- Mock/Real 独立运行，登出与切换账号不会泄漏上一用户缓存或 Realtime 订阅。
- Chrome 和 Android 真机未验证时明确标记待复验，不能宣称发布就绪。
