# 阶段 1 Android 人工验证清单

> 页面开发完成，Web 已检查，Android 原生效果待人工验证。

## 本阶段页面

1. `/empty-state`
2. `/anniversary/countdown`
3. `/data/backup`
4. `/settings`
5. `/ai/comic-result`
6. `/ai/progress`

## 建议测试顺序

1. 从 Dev UI 分别打开六个路由，先确认返回与纵向滚动。
2. 在 `/anniversary/countdown` 切换纪念日与提醒，并打开详情、模板和分享入口。
3. 在 `/data/backup` 执行本地备份、恢复确认、自动备份与频率切换。
4. 在 `/settings` 检查两个 Switch、主题展开、各功能跳转与退出弹窗；退出确认会进入受保护登录页。
5. 在 `/ai/progress` 点击“模拟完成并查看作品”，确认漫画进入 `/ai/comic-result`，视频任务进入 `/ai/video-preview`。
6. 在 `/ai/comic-result` 切换三幕说明、收藏、保存、分享与重新生成。

## Android 重点检查

- Safe Area：刘海/挖孔机型顶部标题不贴边；页面底部按钮不被手势导航条遮挡。
- 阴影与 elevation：卡片边缘应柔和，无旧版灰色圆弧或突兀深色块。
- 字体：中文手写字体加载后不挤压按钮；系统字体放大时标题和说明可换行。
- 底部导航：从 AI 页面返回主 Tab 后，TabBar 不覆盖最后一个操作按钮。
- 图片清晰度：空日记、纪念日、备份、AI 三联漫画在常见 360–430dp 宽度下无拉伸、锯齿和错误裁切。
- 键盘：本阶段页面没有核心文本输入；仅检查弹窗和开关操作不会意外唤起键盘。
- 模糊与渐变：FairyPage 顶部滚动状态应平滑，低端设备上无明显闪白。
- 触控：设置行、分镜卡、主题选项和返回按钮点击区域应稳定，Switch 不应触发父级跳转。

## Web 无法确认

- Android 原生 elevation 与渲染层级。
- Android 字体基线、系统字体缩放和 Ionicons 字体加载时序。
- 手势导航栏、实体返回键及不同厂商 Safe Area 行为。
- 大图在真实设备 GPU 纹理与内存压力下的表现。
