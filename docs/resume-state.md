## 已完成代码范围

### 数据与安全基础

- Supabase 配置、时间戳迁移、核心业务表、RLS、私有 Bucket 和受控 RPC 已存在于分支。
- Mock、Real 和无效 API 模式已明确隔离；未知模式失败关闭，不会回退 Mock。
- App 页面不应直接访问 Supabase Client，统一通过 API/Store。
- Real 模式下未接入的 AI、会员支付、PDF 文件生成、云备份、语音、位置和反馈附件均明确关闭，不模拟成功。

### 认证与会话

- 登录、注册、Session 恢复均验证服务端用户、Session 用户和提交邮箱的一致性。
- Profile 写入限制昵称、头像文字和头像 URL，并验证返回记录属于当前用户。
- 登出、账号变化和本地数据清理会增加 Session epoch、停止 Realtime 并清除内存业务状态。
- 本地 Storage 部分清理失败时仍会退出旧 Session，并返回失败 key。

### 情侣关系

- 邀请码格式和过期时间使用服务端返回真值。
- 绑定结果必须为 active，且当前用户必须是关系成员。
- 非 active、解绑或成员身份异常的关系不会继续暴露伴侣资料或加载情侣业务数据。
- 情侣资料更新结果验证当前用户成员身份。

### 日记与附件

- 日记列表、详情、创建、更新和删除验证情侣空间与作者归属。
- 创建、更新和删除以服务端实际返回行为准，零行写入不会报告成功。
- 附件先上传，数据库失败或异常时仅清理本次新上传对象，不误删调用方已有对象。
- 删除日记后清理本人拥有的附件；清理不完整时明确返回补偿信息。

### 照片集与照片

- 相册列表、详情、创建和删除验证情侣空间、上传者和嵌套照片归属。
- 多图写入数量与返回记录逐项校验。
- 上传失败、数据库失败和异常路径只补偿清理本次新上传对象。
- 删除照片集后清理本人拥有的 Storage 对象，并明确报告部分清理失败。

### 纪念日

- 创建、更新和删除使用服务端返回真值。
- 日期、标题和 ID 有明确输入校验。
- “下一个纪念日”按最近一次实际发生时间计算，兼容每年重复与不重复记录。

### 自定义标签

- 列表、创建、更新和删除验证情侣空间归属。
- 创建结果验证当前创建者。
- 标签名称、分类和图标增加输入边界。

### 时光胶囊

- 读取、创建、更新、删除和提醒 RPC 均要求有效情侣关系。
- 返回记录验证当前情侣空间和目标 ID。
- 创建结果验证当前创建者。
- 未解锁胶囊正文在 API 边界强制清空，防止进入 Store、缓存或调试工具。

### 评论与通知

- 评论目标必须真实存在并属于当前情侣空间。
- 评论列表和创建返回验证情侣空间、目标类型、目标 ID 和作者。
- 评论删除验证当前作者；内容限制 1000 字。
- 通知列表、单项已读和全部已读验证当前用户归属。
- 乐观更新失败只回滚本次目标，不覆盖其他并发成功状态。

### Realtime 与 Store

- Realtime 使用明确业务表白名单，不使用 `table: '*'`。
- 情侣数据按 `couple_id` 或情侣记录 ID 过滤，通知按当前用户 ID 过滤。
- 频道状态统一识别成功、错误、超时和关闭，并接入 Store 错误状态。
- Session 或情侣关系变化后旧频道、旧回调和延迟刷新均被清理或拦截。
- 服务端写入成功但后续刷新失败时，返回 `serverWriteConfirmed` 和 `refreshFailed`，不会误报写入失败导致重复提交。

### 页面真值与能力边界

- AI 视频配置、系统分享、PDF 配置/预览、云备份等页面已完成 Mock/Real 语义隔离。
- 系统分享区分真正分享、用户取消和调用失败。
- PDF 路由参数使用内容类型、封面、纸张和清晰度白名单；Real 模式不创建 Mock 导出任务。
- 云备份页面明确区分 Mock 演示、Real 未开放和无效配置。
- 搜索和时间线继续由已加载业务数据派生，不增加重复业务真相表。
