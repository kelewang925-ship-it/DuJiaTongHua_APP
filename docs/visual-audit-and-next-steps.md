### 2.2 与效果图/规范仍有差距

| 问题 | 当前表现 | 期望方向 |
| --- | --- | --- |
| 底部 Tab 还不够贴纸化 | 当前仍是 Expo Router 默认 TabBar 样式，只做了颜色和阴影调整 | 需要自定义 TabBar，实现激活态桃粉胶囊、图标浮起、贴纸感背景。 |
| 首页记录流偏列表 | `MemoryCard` 逐条纵向排列，缺少“不规则绘本拼贴/回忆碎片墙”效果 | 首页需要增加拼贴式记录流、左右错位卡片、照片贴纸、胶带元素。 |
| 页面纸张纹理不足 | 当前主要依赖纯色背景和卡片阴影 | 需要增加轻量纸张纹理组件或背景装饰，不要过重。 |
| 图标仍偏系统化 | 当前多使用 Ionicons，视觉统一但不够手绘 | 需要补一组项目专属轻手绘图标封装，例如羽毛笔、拍立得、魔法棒、信封、小花。 |
| AI生成页动效不足 | 目前是静态进度条和图标 | 需要加入墨水扩散/星点/纸页浮现等动效或至少模拟动效层。 |
| 弹窗/浮层体系缺失 | 目前没有统一 BottomSheet / Confirm Dialog | 需要补贴纸式弹窗组件，支撑删除确认、AI生成确认、分享面板等。 |
| 账号/引导流程仍需真实化 | 已有页面但未形成完整登录→绑定→主界面状态判断 | 需要增加 auth/binding store 或 mock 登录状态。 |
| 数据持久化缺失 | Zustand 数据刷新后丢失 | 需要接入 AsyncStorage 或 zustand persist。 |
| PNG 效果图未被直接用于像素对齐 | 当前依据设计规范实现，未做逐图对比 | 后续需要按 PNG 中的模块顺序逐页建立视觉验收清单。 |

## 3. 当前项目真实完成度判断

### 3.1 交互原型完成度

当前项目已经不是纯静态页面，已经具备核心本地交互闭环：

```text
写日记 → 首页记录 → 日记详情 → 文本转漫画 → AI生成进度 → 童话工坊
传照片 → 首页记录 → 相册 → 情侣空间动态 → 我的统计
添加纪念日 → 纪念日列表 → 情侣空间动态
生成AI作品 → 创作历史 → 情侣空间动态 → 生成进度页 / 视频预览页
```

### 3.2 视觉还原完成度

按当前 PNG 效果图和设计规范判断：

| 模块 | 完成度 | 说明 |
| --- | --- | --- |

| 核心组件 | 中高 | 卡片、按钮、标签已具备，但缺少输入框、弹窗、底部浮层等组件。 |
| 首页 | 中 | 气质接近，但记录流还不够“拼贴绘本”。 |
| 情侣空间 | 中 | 具备双人动态，但时间轴视觉还可以更手绘化。 |

## 4. 接下来最应该做什么

### 优先级 P0：视觉基础组件补齐

1. **自定义底部 TabBar**
   - 替代默认 TabBar。
   - 实现桃粉激活胶囊、图标微浮起、贴纸式圆角底座。
   - 对齐 `界面设计风格图-2.png` 中底部 Tab 规范。

2. **统一输入框组件 `FairyInput`**
   - 支持单行、多行、前置图标、错误提示。
   - 替换日记、照片、纪念日、AI配置页中的原生 TextInput 样式。

3. **统一弹窗/浮层组件**
   - `FairyDialog`
   - `FairyBottomSheet`
   - `FairyActionSheet`
   - 用于删除确认、分享面板、AI生成确认、退出编辑确认。

4. **统一空状态组件 `FairyEmptyState`**
   - 支持插画、标题、说明、按钮。
   - 用于相册空状态、创作历史空状态、搜索空状态、草稿箱空状态。

---

### 优先级 P1：核心页面高保真还原

1. **首页重排为“回忆碎片墙”**
   - 记录卡片左右错位。
   - 照片记录使用拍立得贴纸样式。
   - 日记记录使用纸页卡片样式。
   - AI作品使用琥珀金点缀。

2. **情侣空间改造成手绘时间轴**
   - 增加竖向曲线或节点线。
   - 节点使用爱心、星星、小花。
   - 动态卡片弱化社交感，强化“第 X 天 · 故事推进”。

3. **童话工坊增强魔法屋视觉**
   - AI漫画和回忆视频入口做成双卡片。
   - 增加星点/金色贴纸/轻渐变。
   - 创作历史做成收藏册样式。

4. **我的页面强化私人收藏册**
   - 会员视觉用琥珀金。
   - 统计卡片做成小贴纸。
   - 功能入口按“纪念日 / 时光胶囊 / 数据导出 / 设置”分组。

---

### 优先级 P2：补完整功能页面

1. 登录 → 绑定判断 → 邀请/确认 → 主界面。
2. 草稿箱、搜索页、分享预览页、会员权益页。
3. 时光胶囊设置页和标签管理页。
4. PDF 导出预览页、存储空间管理页。
5. AI生成成功/失败页。

---

### 优先级 P3：工程化与真实能力

1. 接入 `zustand persist` 或 AsyncStorage，让本地记录不丢失。
2. 抽象 API client，逐步替换 mock 数据。
3. 接入系统相册 ImagePicker。
4. 接入真实 AI 生成接口。
5. 接入文件导出/PDF生成能力。
6. 增加基础表单校验、错误提示和加载状态。

## 5. 推荐下一轮开发顺序

建议下一轮不要继续扩散页面，而是先提升视觉一致性：

```text
1. 自定义底部 TabBar
2. FairyInput / FairyEmptyState / FairyDialog
3. 首页回忆碎片墙重排
4. 情侣空间手绘时间轴
5. 童话工坊魔法感增强
6. zustand persist 数据持久化
```

原因：

- 当前功能闭环已经基本跑通；
- 最大短板已经从“有没有页面”变成“像不像效果图”；
- 先补组件层，后续所有页面都会自然变得更统一；
- 自定义 TabBar 和首页重排是用户最先看到的视觉差异，优先级最高。
