# Fairy 布局系统

> 布局是《独家童话》的空间语法。通过统一的栅格、间距和层级系统，让每一页都像精心排版的绘本一样舒适、有序、有呼吸感。

---

## 1. 设计哲学

### 1.1 空间情绪

- **呼吸感**：大量留白，不拥挤，像翻开一本新书的舒适感
- **层次**：通过间距和阴影建立清晰的信息层级
- **温暖**：圆角、柔和阴影、奶油色背景营造包裹感
- **手账感**：允许轻微打破网格，增加手工温度

### 1.2 布局原则

```
1. 内容优先：留白服务于内容，而非装饰
2. 亲密性：相关元素靠近，无关元素远离
3. 对齐：建立隐形参考线，减少视觉噪音
4. 节奏：重复的模式创造可预测性
5. 打破：偶尔打破规则，增加惊喜和手绘感
```

## 2. 栅格系统（Grid System）

### 2.1 移动端栅格

| 属性 | 规范 | 说明 |
|------|------|------|
| 列数 | 4 列 | 移动端标准 |
| 列宽 | 动态计算 | `(屏幕宽度 - 边距×2 - 间距×3) / 4` |
| 间距 (gutter) | 16px | 列与列之间 |
| 边距 (margin) | 16px（标准）/ 20px（宽松） | 屏幕两侧 |

### 2.2 断点规范

| 断点 | 宽度 | 布局调整 |
|------|------|----------|
| `xs` | < 360px | 紧凑模式，边距 12px |
| `sm` | 360px ~ 414px | 标准模式，边距 16px |
| `md` | 414px ~ 768px | 宽松模式，边距 20px |
| `lg` | > 768px | 平板模式，边距 24px，可扩展为 6 列 |

### 2.3 栅格使用规则

- **卡片宽度**：1 列（小卡片）、2 列（标准卡片）、4 列（全宽卡片）
- **按钮宽度**：根据内容自适应，最小 2 列宽
- **表单输入**：全宽（4 列）
- **图片**：可跨 2 列或 4 列

## 3. 间距系统（Spacing System）

### 3.1 间距 Token

基于 4px 基准，建立 8 级间距：

| Token | 值 | 用途 |
|-------|-----|------|
| `space-0` | 0px | 无间距 |
| `space-xs` | 4px | 图标与文字间距、紧凑内联 |
| `space-sm` | 8px | 小元素间距、按钮内边距 |
| `space-md` | 12px | 列表项间距、卡片内部小间距 |
| `space-lg` | 16px | 标准间距、卡片内边距 |
| `space-xl` | 20px | 模块间距、页面边距 |
| `space-2xl` | 24px | 大模块间距、卡片之间 |
| `space-3xl` | 32px | 页面模块间距 |
| `space-4xl` | 48px | 大区块间距、空状态 |

### 3.2 间距使用模式

| 场景 | 间距组合 | 说明 |
|------|----------|------|
| 页面边距 | `space-xl` (20px) | 标准页面 |
| 卡片内边距 | `space-lg` (16px) | 标准卡片 |
| 大卡片内边距 | `space-xl` (20px) | 内容较多卡片 |
| 卡片间距 | `space-md` (12px) | 卡片列表 |
| 列表项间距 | `space-sm` (8px) | 紧凑列表 |
| 表单字段间距 | `space-lg` (16px) | 表单布局 |
| 按钮组间距 | `space-sm` (8px) | 并排按钮 |
| 模块间距 | `space-2xl` (24px) | 页面区块 |

## 4. 安全区域（Safe Area）

### 4.1 系统安全区域

| 区域 | 规范 | 说明 |
|------|------|------|
| 顶部 | 状态栏高度 + 8px | iOS 刘海屏需额外处理 |
| 底部 | Home Indicator + 16px | iOS 底部横条 |
| 侧边 | 边距（16px 或 20px） | 避免贴边 |

### 4.2 内容安全区域

```
┌─────────────────────────────┐
│  顶部安全区 (状态栏 + 8px)   │
├─────────────────────────────┤
│                             │
│      内容区域               │
│      (边距 16px)            │
│                             │
├─────────────────────────────┤
│  底部安全区 (Home条 + 16px) │
└─────────────────────────────┘
```

## 5. 圆角系统（Radius System）

### 5.1 圆角 Token

| Token | 值 | 用途 |
|-------|-----|------|
| `radius-none` | 0px | 极少数场景（如全屏图片） |
| `radius-sm` | 8px | 小按钮、标签、输入框 |
| `radius-md` | 12px | 标准按钮、小卡片 |
| `radius-lg` | 16px | 标准卡片、弹窗 |
| `radius-xl` | 20px | 大卡片、模块容器 |
| `radius-2xl` | 24px | 特殊卡片、图片容器 |
| `radius-full` | 9999px | 圆形元素（头像、FAB） |

### 5.2 圆角使用规则

- **卡片**：`radius-lg` (16px) 或 `radius-xl` (20px)
- **按钮**：`radius-md` (12px) 或 `radius-lg` (16px)
- **输入框**：`radius-sm` (8px) 或 `radius-md` (12px)
- **头像**：`radius-full` (圆形)
- **标签/徽章**：`radius-full` (胶囊形)
- **弹窗**：`radius-xl` (20px)

### 5.3 特殊圆角

- **贴纸感**：允许轻微旋转 `-2° ~ 2°` + 圆角卡片
- **相框感**：内圆角 + 外边框，营造复古相框效果

## 6. 阴影系统（Shadow System）

### 6.1 阴影 Token

| Token | 参数 | 用途 |
|-------|------|------|
| `shadow-sm` | y: 2, blur: 8, opacity: 0.04 | 轻微浮起，标签、小按钮 |
| `shadow-md` | y: 6, blur: 18, opacity: 0.08 | 标准卡片阴影 |
| `shadow-lg` | y: 12, blur: 30, opacity: 0.12 | 浮动元素、FAB |
| `shadow-xl` | y: 20, blur: 50, opacity: 0.16 | 弹窗、模态框 |

### 6.2 阴影颜色

- **默认**：`cocoa-300` (可可棕) 为基础色
- **桃粉变体**：`pink-400` 为基础色，用于桃粉卡片
- **禁止**：纯黑阴影（#000000），必须使用暖色调阴影

### 6.3 阴影使用规则

| 场景 | 阴影 | 说明 |
|------|------|------|
| 卡片默认 | `shadow-md` | 轻微浮起，区分层级 |
| 卡片按下 | `shadow-sm` | 按下时阴影缩小，模拟按压 |
| 浮动按钮 | `shadow-lg` | 明显浮起，吸引注意力 |
| 弹窗 | `shadow-xl` | 最强阴影，覆盖全屏 |
| 输入框聚焦 | `shadow-sm` + border-focus | 轻微 glow 效果 |

## 7. 层级系统（Z-Index）

### 7.1 层级定义

| 层级 | Z-Index | 用途 |
|------|----------|------|
| 背景层 | 0 | 页面背景、底层内容 |
| 内容层 | 10 | 标准内容、卡片 |
| 浮动元素 | 50 | FAB、浮动按钮 |
| 抽屉/侧边栏 | 80 | 侧边导航、筛选面板 |
| 弹窗/模态框 | 100 | Dialog、底部弹窗 |
| 遮罩层 | 150 | 半透明遮罩 |
| 全局提示 | 200 | Toast、Snackbar |
| 加载层 | 300 | 全屏加载、遮罩 |

## 8. 布局模式

### 8.1 标准页面布局

```
┌─────────────────────────────┐
│  Header (FairyHeader)       │
│  - 返回按钮 / 标题 / 操作    │
├─────────────────────────────┤
│                             │
│  Content Area               │
│  - 滚动内容                 │
│  - 卡片列表 / 表单 / 详情   │
│                             │
├─────────────────────────────┤
│  Bottom Action (可选)        │
│  - 主要操作按钮              │
└─────────────────────────────┘
```

### 8.2 卡片列表布局

```
┌─────────────────────────────┐
│  [卡片 1]  全宽              │
│  ─────────────────────────  │
│  [卡片 2]  全宽              │
│  ─────────────────────────  │
│  [卡片 3]  全宽              │
└─────────────────────────────┘
间距：卡片之间 12px (space-md)
```

### 8.3 双列网格布局

```
┌─────────────────────────────┐
│  [卡片 1]  │  [卡片 2]      │
│  ──────────┼───────────    │
│  [卡片 3]  │  [卡片 4]      │
└─────────────────────────────┘
间距：列间距 12px，行间距 12px
```

### 8.4 表单布局

```
┌─────────────────────────────┐
│  标签                        │
│  ┌───────────────────────┐  │
│  │  输入框内容             │  │
│  └───────────────────────┘  │
│  辅助说明 / 错误提示          │
│                              │
│  标签                        │
│  ┌───────────────────────┐  │
│  │  输入框内容             │  │
│  └───────────────────────┘  │
└─────────────────────────────┘
间距：标签与输入框 4px，字段之间 16px
```

## 9. AI 生成规则

1. **使用 Token**：间距必须使用 space-* Token，禁止硬编码 13px、17px 等值
2. **边距一致**：同一页面边距必须统一（16px 或 20px）
3. **圆角统一**：同一层级元素圆角必须一致
4. **阴影柔和**：阴影必须使用暖色调，禁止纯黑
5. **安全区域**：必须考虑顶部状态栏和底部 Home Indicator
6. **响应式**：布局必须适配 xs/sm/md 三种断点
