# Voice and Tone

> 文案是《独家童话》的灵魂。每一句话都应该像恋人在耳边低语，温暖、真诚、有温度。

---

## 1. 语气定义

### 1.1 核心语气

| 语气 | 特征 | 使用场景 |
|------|------|----------|
| **温暖** | 像阳光照进房间，让人感到被爱 | 欢迎、成功、日常问候 |
| **俏皮** | 偶尔的小幽默，像恋人的玩笑 | 新功能、彩蛋、轻量提示 |
| **鼓励** | 失败时的正向引导，像朋友的安慰 | 错误、空状态、重试 |
| **神秘** | 新功能预告时的期待感 | 功能未开放、VIP 预告 |
| **浪漫** | 像情诗一样优美 | 纪念日、特殊时刻 |

### 1.2 语气矩阵

| 场景 | 语气 | 示例 |
|------|------|------|
| 成功 | 庆祝、温暖 | "回忆已收藏进故事书~" |
| 错误 | 安慰、鼓励 | "小信使迷路了，再试一次吧~" |
| 加载 | 神秘、期待 | "魔法画笔正在悄悄涂色..." |
| 空状态 | 温柔、引导 | "这里还没有故事呢，来写第一篇吧~" |
| 确认 | 谨慎、关怀 | "要把这段回忆藏进小黑屋吗？" |
| 引导 | 热情、邀请 | "把今天写进童话~" |
| 告别 | 温柔、不舍 | "明天见，记得想我哦~" |
| 欢迎 | 热情、温暖 | "欢迎回来，今天也被好好爱着。" |

## 2. 禁用词汇

### 2.1 技术词汇（绝对禁止）

| 禁用 | 原因 | 替换 |
|------|------|------|
| "系统" | 冰冷、机械 | "故事书"、"童话世界" |
| "服务器" | 技术感 | "小信使的家"、"邮局" |
| "数据库" | 技术感 | "记忆仓库"、"故事书架" |
| "网络" | 抽象 | "小路"、"桥梁" |
| "错误码" | 无意义 | 隐藏，用角色表达 |
| "异常" | 负面 | "小插曲"、"小意外" |
| "故障" | 严重 | "故事书暂时打不开了" |
| "Loading..." | 英文、无情感 | "魔法正在发生..." |
| "Processing..." | 英文、无情感 | "小精灵正在整理..." |
| "请" | 过于正式 | 直接说，或加"~" |
| "务必" | 命令式 | 引导式表达 |
| "必须" | 命令式 | "需要"、"可以" |
| "点击这里" | 过时 | 直接说操作 |
| "请点击此处" | 过时、正式 | 直接说操作 |

### 2.2 负面词汇（尽量避免）

| 禁用 | 替换 |
|------|------|
| "失败" | "还没成功"、"差一点" |
| "错误" | "小插曲"、"意外" |
| "无法" | "暂时不能"、"还没办法" |
| "禁止" | "还不能"、"需要等一等" |
| "删除" | "藏进小黑屋"、"收起来" |

## 3. 命名规范

### 3.1 功能命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 动词 + 名词 | 温暖、有画面感 | "收藏回忆"、"书写日记"、"开启童话" |
| 避免 | 纯技术动词 | "提交表单"、"执行操作" |

### 3.2 状态命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 形容词 + 名词 | 有故事感 | "魔法加载中"、"回忆整理中"、"故事成型中" |
| 避免 | 纯技术状态 | "处理中"、"加载中" |

### 3.3 按钮命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 动词 | 简洁、有行动感 | "保存回忆"、"开始创作"、"邀请 TA" |
| 避免 | 名词或形容词 | "确定"、"好的"（太冷淡） |

## 4. 特殊场景文案

### 4.1 欢迎语

```
首次打开：
  "欢迎来到独家童话，这里将记录你们的故事。"

每日欢迎：
  "欢迎回来，今天也被好好爱着。"
  "今天想记录什么故事呢？"
  "把今天写进童话~"
```

### 4.2 告别语

```
退出编辑：
  "故事还没写完，真的要离开吗？"

退出 App：
  "明天见，记得想我哦~"
```

### 4.3 节日/特殊时刻

```
纪念日：
  "今天是你们在一起的第 X 天，值得被好好纪念。"

生日：
  "祝 TA 生日快乐！今天的故事一定很特别~"

情人节：
  "情人节快乐，今天的故事书会特别甜~"
```

## 5. 文案检查清单

发布前检查：

- [ ] 是否使用了禁用词汇？
- [ ] 是否符合当前场景的语气？
- [ ] 是否温暖、有情感？
- [ ] 是否简洁、不啰嗦？
- [ ] 是否有画面感？
- [ ] 是否适合朗读？
- [ ] 是否考虑了情侣双方的感受？

---

## 6. AI 生成规则

1. **先确定语气**：根据场景选择语气，再写文案
2. **检查禁用词**：生成后检查是否包含禁用词汇
3. **朗读测试**：读出来，是否像恋人在说话？
4. **情感优先**：功能准确第二，情感表达第一
5. **一致性**：同一功能的文案在不同页面必须一致
