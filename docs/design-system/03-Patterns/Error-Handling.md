# Error Handling Patterns

> 错误处理模式。像童话书中的小挫折，温柔安慰，引导解决。

---

## 1. 错误类型

### 1.1 网络错误

```
[小信使迷路插画]
"小信使迷路了，再试一次吧~"
[重试按钮]
```

- 角色：小信使（迷糊表情）
- 文案：温柔、不指责
- 操作：重试按钮
- 自动重试：可选，最多 3 次

### 1.2 服务器错误

```
[伤心小熊插画]
"故事书暂时打不开了，稍后再试吧~"
[刷新按钮] [返回首页]
```

- 角色：伤心小熊
- 文案：解释 + 安慰
- 操作：刷新或返回

### 1.3 表单错误

```
[输入框摇晃动画]
"我们的暗号太短啦，再多写几个字吧"
```

- 视觉：输入框边框变红 + 轻微摇晃
- 文案：具体 + 情绪化
- 操作：自动聚焦到错误字段

### 1.4 权限错误

```
[独角兽插画]
"这个功能需要独角兽的许可哦~"
[去设置] [知道了]
```

- 角色：独角兽（神秘表情）
- 文案：引导用户授权
- 操作：跳转到设置

---

## 2. 错误文案规范

| 场景 | 禁止 | 推荐 |
|------|------|------|
| 网络 | 网络连接失败 | 小信使迷路了，再试一次吧~ |
| 服务器 | 服务器错误 500 | 故事书暂时打不开了，稍后再试吧~ |
| 表单 | 密码长度不足 | 我们的暗号太短啦，再多写几个字吧 |
| 权限 | 权限不足 | 这个功能需要独角兽的许可哦~ |
| 超时 | 请求超时 | 小信使飞得太慢了，再试一次吧~ |
| 不存在 | 资源不存在 | 这段回忆被小精灵藏起来了~ |

---

## 3. 错误恢复

### 3.1 自动恢复

- 网络错误：自动重试 3 次，间隔 2s、4s、8s
- 加载失败：自动刷新（可选）

### 3.2 手动恢复

- 重试按钮：最常用
- 刷新按钮：页面级错误
- 返回按钮：深层导航错误
- 设置按钮：权限错误

---

## 4. AI 生成规则

1. 错误必须有角色插画，禁止纯错误码
2. 文案必须安慰 + 解决建议，禁止仅描述错误
3. 必须提供恢复操作，不能让用户卡住
4. 技术错误码（如 500）必须隐藏，不展示给用户
