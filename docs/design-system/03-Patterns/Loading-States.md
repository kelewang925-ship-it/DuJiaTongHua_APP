# Loading State Patterns

> 加载状态模式。像等待童话故事的下一页，有期待感、不焦虑。

---

## 1. 加载类型

### 1.1 全屏加载

```
┌─────────────────────────────┐
│                              │
│      [角色动画]               │
│      魔法画笔正在悄悄涂色...   │
│      [进度条]                 │
│                              │
└─────────────────────────────┘
```

- 背景：bg-primary
- 角色：120px，循环动画
- 文案：故事化，逐字显示或循环切换
- 进度条：可选
- 禁止："Loading..."、旋转菊花（无情感）

### 1.2 局部加载

```
┌─────────────────────────────┐
│  [内容区域]                  │
│  [骨架屏 / 占位卡片]          │
│  [骨架屏 / 占位卡片]          │
└─────────────────────────────┘
```

- 骨架屏：奶油色块，shimmer 动画
- 占位卡片：FairyCard 样式，内容区域为灰色块
- 禁止：空白或 "加载中..." 文字

### 1.3 下拉刷新

```
┌─────────────────────────────┐
│  ↓ 松开刷新                  │
│  🔄 小信使正在整理...         │
│  ✓ 回忆已更新                │
└─────────────────────────────┘
```

- 指示器：小信使或角色动画
- 文案：根据状态变化
- 成功反馈：轻微震动 + 文案

### 1.4 上拉加载更多

```
┌─────────────────────────────┐
│  [内容列表]                  │
│  ─────────────────────────  │
│  🔄 正在加载更多回忆...      │
└─────────────────────────────┘
```

- 底部指示器：小精灵或旋转动画
- 文案："正在加载更多回忆..."
- 无更多："所有回忆都加载完了~"

## 2. 角色加载动画

### 2.1 小信使（网络请求）

```
动画：飞行轨迹，翅膀扇动
文案：
  - "小信使正在赶路..."
  - "小信使飞过山川河流..."
  - "信件正在穿越云层..."
```

### 2.2 魔法画笔（AI 生成）

```
动画：旋转 + 笔尖发光 + 颜料飞溅
文案：
  - "魔法画笔正在悄悄涂色..."
  - "彩色铅笔在纸上跳舞..."
  - "故事书正在一页页成型..."
```

### 2.3 小精灵（保存/收藏）

```
动画：装订书页 + 撒星星
文案：
  - "回忆正在藏进故事书..."
  - "小精灵正在装订书页..."
  - "这段故事已被温柔收藏..."
```

## 3. 进度表达

### 3.1 百分比进度

```
[=====>    ] 50%
"已完成一半 - 故事即将成型..."
```

### 3.2 步骤进度

```
步骤 1/4：收集回忆 ✓
步骤 2/4：魔法涂色 → 进行中
步骤 3/4：装订成书
步骤 4/4：交付给你
```

### 3.3 时间预估

```
"大约还需要 30 秒..."
"故事即将完成，请稍候..."
```

---

## 4. AI 生成规则

1. 加载必须有角色动画，禁止纯技术加载图标
2. 文案必须故事化，禁止 "Loading..."
3. 进度必须可感知，禁止无限等待
4. 加载超过 5 秒必须提供取消选项
5. 加载失败必须有错误状态和重试按钮
