# Navigation Patterns

> 导航模式。像童话书中的目录，清晰、温暖、引导探索。

---

## 1. 底部导航（Bottom Tabs）

### 1.1 结构

```
┌─────────────────────────────┐
│  [首页]  [情侣]  [工坊]  [我的] │
│   🏠      💑      ✨      👤   │
│   记录    空间    魔法    个人   │
└─────────────────────────────┘
```

### 1.2 规范

| 属性 | 规范 |
|------|------|
| 高度 | 64px（含安全区域）|
| 背景 | bg-tertiary + 顶部 1px 分割线 |
| 图标 | 24px，未选中 text-secondary，选中 text-accent |
| 文字 | caption (12px)，未选中 text-secondary，选中 text-accent |
| 动效 | 选中时图标轻微放大 1.1 + 颜色过渡 200ms |
| 徽章 | 右上角小红点或数字徽标 |

### 1.3 交互

- 点击：切换页面，haptics light
- 长按：预览（可选）
- 未读消息：徽标 + 徽标动画

---

## 2. 顶部导航（Header）

### 2.1 标准 Header

```
┌─────────────────────────────┐
│  ← 返回    页面标题    [操作]  │
└─────────────────────────────┘
```

| 属性 | 规范 |
|------|------|
| 高度 | 56px |
| 背景 | bg-primary 或透明 |
| 标题 | h1 (24px, bold)，居中或左对齐 |
| 返回按钮 | 24px 图标 + "返回" 文字（可选）|
| 操作按钮 | 24px 图标或文字按钮 |

### 2.2 大标题 Header

```
┌─────────────────────────────┐
│  欢迎回来 👋                 │
│  今天也被好好爱着。           │
│                              │
│  [搜索栏]                     │
└─────────────────────────────┘
```

| 属性 | 规范 |
|------|------|
| 标题 | display (32px, bold) |
| 副标题 | body (14px, secondary) |
| 搜索栏 | 全宽，圆角 12px |

---

## 3. 返回与面包屑

### 3.1 返回按钮

```
[←] 返回
或
[←]
```

- 图标：24px，左箭头
- 文字：caption (12px)，可选
- 点击区域：44×44px（最小触控区域）
- 动效：按下 scale 0.9

### 3.2 面包屑（少用）

```
首页 > 日记 > 详情
```

- 仅在深层导航使用
- 文字：caption (12px)
- 当前页：text-primary，其他：text-secondary + 可点击

---

## 4. 抽屉导航（Drawer）

### 4.1 规范

| 属性 | 规范 |
|------|------|
| 宽度 | 屏幕宽度的 70%，最大 320px |
| 背景 | bg-tertiary |
| 遮罩 | surface-overlay |
| 动画 | translateX: -100% → 0，300ms，decelerate |
| 关闭 | 点击遮罩或右滑 |

---

## 5. AI 生成规则

1. 底部导航必须使用 Fairy 图标风格
2. Header 标题必须情绪化，禁止纯功能命名
3. 返回按钮必须有足够点击区域（44×44px）
4. 导航切换必须有动效，不能生硬跳转
