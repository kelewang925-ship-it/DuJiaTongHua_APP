# 第二阶段批次记录：Real 数据持久化复验

## 触发

重新登录后，页面初始看似未出现先前联调的日记和图片，需要确认是否为数据丢失。

## 实际结果

- 本机仍以 `EXPO_PUBLIC_API_MODE=real` 运行，Supabase URL 与 Anon Key 均已配置。
- 两份已绑定浏览器会话均显示相同情侣关系和 `记录 6 条`：5 篇日记、1 组照片。
- `/diary` 完成加载后显示 5 篇既有日记，时间为 `2026/07/18`，证明列表使用真实 `createdAt`。
- `/photo/album` 完成加载后显示 1 组“Storage 验证”照片和 1 张已记录照片。

## 结论

数据未删除，也不是 Mock/Real 模式切换或关系隔离导致。应用启动时会清空上一会话页面缓存并异步加载情侣模块；本轮实际加载约 15 秒，加载完成前容易被误判为数据缺失。

该加载体验需保留为后续性能优化项；本批不将其伪报为数据持久化失败。
