# 第二阶段批次：首页模块加载空状态修复

## 问题与根因

已登录的 Real 会话在启动时先完成关系数据，再异步拉取日记、相册、纪念日、标签、胶囊与通知。Store 在这段时间使用 `loading.modules` 标识请求中，但首页此前只读取 `loading.bootstrap`，所以会把尚未返回的模块数据渲染为“0 条记录 / 0 篇日记 / 0 张照片”，容易被误判为测试数据丢失。

## 修复

- 首页请求状态同时监听 `loading.bootstrap` 与 `loading.modules`。
- 首页错误状态同时读取模块加载错误，保留既有重试入口。
- Real 模块仍在请求时展示请求态，不渲染业务空状态；请求完成后再展示真实统计与记录。

## 验证

- 新增首页静态回归测试，先在旧实现上失败，再在修复后通过。
- `npm run test:final`：93 个测试套件、304 个测试全部通过；四项页面、Real 能力、Real 页面和 API 审计均通过。
- 已用已绑定的真实浏览器会话刷新首页，恢复后显示 6 条记录、5 篇日记和 1 组照片，确认此前数据没有丢失。

## 后续边界

首屏到核心模块完成所需时间仍可能受网络与 Supabase 响应影响；加载性能与加载层视觉设计继续属于第四阶段优化项。本批仅保证加载期间不展示误导性的业务空数据。
