# 2026-07-18 首屏渐进加载修复

## 范围

- 分支：`codex/phase2-real-data`
- 关联台账：P4-01
- 不修改 Supabase 迁移、RLS、Storage 规则或第二阶段业务范围。

## 根因

Web 根布局把约 10 MB 自定义字体作为路由挂载前置条件；同时 Real 模式启动在离开 `AuthGate` 全屏启动层前等待全部首页模块。慢网络、某个模块失败或字体资源未完成时，用户会长时间看到“正在准备/打开你们的童话…”。

## 修复

- Web 首屏使用系统字体，不再由自定义字体阻塞 Expo Router 挂载；原生端仍注册项目字体。
- 启动只等待已验证 Session 和当前情侣关系；日记、相册、纪念日、标签、胶囊与通知改为后台加载并使用 `modules` 状态。
- 关系请求限制为 5 秒；身份校验限制为 4 秒，`AuthGate` 增加 5 秒总上限，异常时不再无限显示全屏加载层。
- 添加字体降级、渐进式加载和启动超时静态回归测试。

## 验证

- 定向 Jest：5 个套件、16 项通过。
- `npm run audit:pages`、`npm run audit:real`、`npm run audit:real-pages`、`npm run audit:api` 通过。
- 已连接 Chrome 的真实登录会话更新后直接渲染首页，未显示启动层。

- Android 真机冷启动量测与弱网基线仍待做；本修复不构成发布就绪结论。
