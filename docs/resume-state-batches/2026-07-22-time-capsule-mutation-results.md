# 2026-07-22 时光胶囊变更结果契约

代码复核发现：删除和提醒 RPC 原先返回 `void`，而客户端需要服务端返回的胶囊 ID 判断真实成功，存在“数据库已变更但页面报失败”的风险。

新增 `202607220011_time_capsule_mutation_results.sql`：

- 删除 RPC 返回被删除的真实胶囊行；
- 提醒 RPC 返回更新后的真实胶囊行；
- 找不到胶囊或当前情侣无权操作时抛出异常；
- 保留 SECURITY DEFINER、active 情侣校验和 authenticated-only 执行权限。

执行迁移后再验证 A/B 的提醒切换、A 删除自己的胶囊、B 删除 A 胶囊被拒绝，以及失败时页面提示。
